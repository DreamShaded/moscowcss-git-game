# `HEAD@{n}` vs `HEAD~n` vs `HEAD^n` — три способа указать на предка

Три синтаксиса выглядят похоже, но указывают на разные вещи. Путаница между ними — один из самых частых источников неожиданных результатов в git.

## Разница

**`HEAD~n`** — n коммитов назад по *первому родителю*. `HEAD~1` — родитель текущего коммита, `HEAD~3` — прадедушка. Это то, что большинство людей имеют в виду, когда говорят "N коммитов назад".

**`HEAD^n`** — n-й *родитель* коммита (не n шагов назад). Актуально для merge-коммитов, у которых два родителя: `HEAD^1` — первый родитель (ветка, в которую мержили), `HEAD^2` — второй (ветка, которую мержили). Для обычных коммитов `HEAD^` и `HEAD^1` — то же, что `HEAD~1`.

**`HEAD@{n}`** — позиция HEAD n записей назад в **reflog**. Это не про структуру дерева коммитов, а про *историю переключений*: коммитов, checkout, reset. `HEAD@{1}` — где HEAD был до последней операции.

## Наглядно

```
A - B - C - D  ← HEAD
            |
         HEAD~0 = HEAD = HEAD^0
         HEAD~1 = HEAD^ = D's parent (C)
         HEAD~2 = B
         HEAD~3 = A

Merge-коммит M (родители: X и Y):
         M^1 = X  (первый родитель, "наша" ветка)
         M^2 = Y  (второй родитель, влитая ветка)
         M~1 = M^1 = X
```

```bash
git show HEAD~3          # коммит три шага назад
git show HEAD^2          # второй родитель merge-коммита
git show HEAD@{2}        # где HEAD был позавчера (по reflog)
git diff HEAD~1 HEAD     # изменения в последнем коммите
```

## Подвохи

- **`HEAD@{n}` зависит от твоих действий**, а не от структуры репо. У коллеги `HEAD@{1}` будет другим.
- **`HEAD^2` работает только на merge-коммитах.** На обычном коммите `HEAD^2` — ошибка.
- Можно комбинировать: `HEAD~2^2` — второй родитель коммита, который на два шага назад по первому родителю.
- В shell некоторые символы (`^`, `~`) надо экранировать или брать в кавычки в зависимости от оболочки. В PowerShell `^` — специальный символ.
