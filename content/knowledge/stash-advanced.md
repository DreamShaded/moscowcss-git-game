# `git stash` — продвинутые флаги

Базовый `git stash` прячет tracked изменения. Но у него есть флаги для менее очевидных ситуаций: включить untracked файлы, оставить индекс нетронутым, сохранить только часть изменений, или превратить stash в ветку.

## Флаги

**`-u` / `--include-untracked`** — включить в stash неотслеживаемые файлы (те, которых нет в `git status` как "new file"). Без этого флага новые файлы остаются в worktree.

**`-a` / `--all`** — включить и untracked, и игнорируемые файлы (из `.gitignore`). Используй осторожно — можно спрятать node_modules.

**`-k` / `--keep-index`** — спрятать только то, что *не* в индексе. Проиндексированное останется готовым к коммиту. Полезно когда сделал `git add` и хочешь спрятать "всё остальное".

**`-p` / `--patch`** — интерактивный выбор hunks для stash. Подробнее: [`stash-patch`](./stash-patch.md).

**`push -m "описание" -- path/to/file`** — спрятать только конкретный файл или директорию с описанием.

## Примеры

```bash
git stash -u                          # включить untracked файлы
git stash -k                          # спрятать всё кроме индекса
git stash push -m "wip: modal styles" -- src/styles/modal.css

# Работа со списком stash:
git stash list                        # показать все stash-записи
git stash show stash@{2}              # посмотреть конкретный stash
git stash pop stash@{2}               # применить конкретный stash
git stash drop stash@{2}              # удалить stash без применения

# Создать ветку из stash:
git stash branch feature/modal-wip stash@{0}
```

## `stash branch` — недооценённая команда

```bash
git stash branch <new-branch> [stash]
```

Создаёт новую ветку на коммите, где был создан stash, применяет stash и удаляет его. Полезно когда понял, что stash вырос в реальную фичу.

## Подвохи

- **Stash хранится глобально для репо** — виден из всех веток. Можно случайно применить stash из одной ветки в другой.
- **`stash pop` = `stash apply` + `stash drop`.** Если `apply` прошёл с конфликтом — stash не удаляется автоматически. `pop` в этом случае тоже не удалит.
- **`-u` без `-a` не берёт ignored файлы.** Если нужны именно ignored — используй `-a`, но с пониманием что туда может попасть много лишнего.
- **Stash — не замена коммиту** для долгосрочного хранения. При gc сташи не удаляются (они достижимы), но лучше не полагаться на это.
