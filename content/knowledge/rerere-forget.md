# `git rerere forget` — удалить ошибочно записанное разрешение конфликта

Если `git rerere` записал неправильное разрешение конфликта — его можно удалить через `git rerere forget`. Иначе git будет продолжать применять неверное решение автоматически при каждом аналогичном конфликте.

## Когда это нужно

- Разрешил конфликт неправильно, сделал `git add` — rerere запомнил ошибку.
- Код изменился, и старое разрешение конфликта больше не актуально, но паттерн конфликта остался тем же.
- Хочешь "сбросить" автоматику для конкретного файла и разрешить конфликт вручную заново.

## Как использовать

```bash
# Удалить записанное разрешение для конкретного файла
git rerere forget <file>
git rerere forget src/styles/theme.css

# Посмотреть что записано в rr-cache
git rerere status                    # файлы с записанными решениями
ls .git/rr-cache/                    # сами записи (по хешу конфликтного паттерна)

# Удалить все записи (ядерный вариант)
rm -rf .git/rr-cache/
```

## Подвохи

- **`git rerere forget` работает только пока конфликт активен.** Если merge уже завершён — файл больше не в "конфликтном" состоянии, и команда не найдёт что забывать. В таком случае — вручную удалять из `.git/rr-cache/`.
- **Запись в rr-cache идентифицируется хешем** конфликтного паттерна, а не именем файла. Если конфликт немного другой (изменился контекст вокруг маркеров) — это уже другая запись.
- **После `forget`** при следующем конфликте rerere будет записывать разрешение заново с нуля.
- **Ручная чистка `.git/rr-cache/`** безопасна — это только кэш. Худшее что случится: rerere не сможет автоматически разрешить конфликты, которые мог бы.
