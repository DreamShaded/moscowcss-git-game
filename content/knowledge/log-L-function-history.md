# `git log -L` — история конкретного блока кода или функции

`git log -L` отслеживает эволюцию конкретного диапазона строк или именованной функции через всю историю. Вместо поиска по ключевому слову — точная история того, как менялся этот конкретный кусок кода.

Для фронтендера это особенно полезно: можно отследить историю CSS-правила, функции, или блока — даже если файл при этом переименовывался или реорганизовывался.

## Как использовать

```bash
# История строк с 10 по 25 в файле:
git log -L 10,25:src/styles/button.css

# История от строки 10 до конца функции (автодетект по { }):
git log -L 10,+20:src/styles/button.css

# История по имени функции (работает для JS, Python, C и др.):
git log -L :functionName:src/utils.js

# Для CSS-блока — по строке начала:
git log -L '/\.btn-primary/,/^}/:src/styles/button.css'
```

## Что показывает вывод

Каждый коммит в выводе содержит только diff именно этого диапазона строк — не весь коммит, а только изменения в отслеживаемом блоке. Очень удобно для code archaeology.

```bash
# Пример: кто и когда трогал z-index модального окна
git log -L '/\.modal/,/^}/:src/styles/modal.css' --oneline
```

## Подвохи

- **Для CSS синтаксис функций не работает напрямую.** `git log -L :selector:file.css` не знает CSS-селекторы. Используй диапазоны строк или regex-диапазон `/start/,/end/`.
- **Переименование файла** прерывает историю. Добавь `-M` для отслеживания переименований: `git log -L 10,25:new-name.css -M`.
- **Медленнее обычного log** на больших репо — `-L` проходит через весь граф коммитов.
- **`-L` и `-p`** совместимы: `git log -L 10,25:file.css -p` добавит полный diff коммита.
