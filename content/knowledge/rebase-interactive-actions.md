# Действия в интерактивном rebase: pick, squash, fixup, edit, drop, reword

`git rebase -i HEAD~N` открывает редактор со списком N последних коммитов. Перед каждым — команда (действие). Можно менять порядок строк и команды, чтобы переписать историю.

## Таблица команд

| Команда | Сокращение | Что делает |
|---------|-----------|-----------|
| `pick` | `p` | Оставить коммит как есть (дефолт) |
| `reword` | `r` | Оставить коммит, но изменить сообщение |
| `edit` | `e` | Остановиться на этом коммите для изменений |
| `squash` | `s` | Слить с предыдущим коммитом, объединить сообщения |
| `fixup` | `f` | Слить с предыдущим коммитом, выбросить сообщение |
| `drop` | `d` | Удалить коммит из истории |
| `exec` | `x` | Выполнить shell-команду между коммитами |

## Примеры

```
# Исходный список:
pick abc123 feat: add button component
pick def456 wip: styles
pick ghi789 wip: fix styles
pick jkl012 fix: correct hover state

# После редактирования — squash wip-коммитов:
pick abc123 feat: add button component
squash def456 wip: styles
fixup ghi789 wip: fix styles
reword jkl012 fix: correct hover state
```

Результат: два коммита. Первый объединит abc123+def456+ghi789 (с возможностью написать новое сообщение). Второй позволит переименовать сообщение jkl012.

## Как использовать `edit`

```bash
# В списке поставить edit перед нужным коммитом.
# Rebase остановится на нём. Тогда:
git add <file>               # добавить забытый файл
git commit --amend           # изменить коммит
git rebase --continue        # продолжить
```

## Подвохи

- **Порядок строк = порядок коммитов.** Перестановка строк — это перестановка коммитов в истории. Ошибка в порядке — конфликты.
- **`squash` и `fixup` применяются к предыдущему коммиту** (строкой выше), не к следующему. Нельзя поставить squash на первую строку.
- **`drop` удаляет коммит**, но не файлы из рабочей директории — только коммит из истории. Если в нём были нужные изменения — они потеряются (до gc).
- **Отменить можно через** `git rebase --abort` (если ещё идёт) или `git reset --hard ORIG_HEAD` (после завершения).
- **`--autosquash`** автоматически расставляет `fixup`/`squash` для коммитов, созданных через `git commit --fixup`. Подробнее: [`commit-fixup-flow`](./commit-fixup-flow.md).
