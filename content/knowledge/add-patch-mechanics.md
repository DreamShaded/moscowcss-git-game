# `git add -p` — интерактивное индексирование по кускам

`git add -p` (или `--patch`) позволяет выбрать, какие именно изменения в файле попадут в следующий коммит. Git разбивает изменения на "hunks" (куски) и спрашивает по каждому: добавить или нет. Незакоммиченные части остаются в worktree.

Это главный инструмент для создания чистых, атомарных коммитов когда в одном файле перемешаны несвязанные изменения.

## Буквы-команды в диалоге

| Команда | Действие |
|---------|---------|
| `y` | Добавить этот hunk в индекс |
| `n` | Пропустить этот hunk |
| `s` | Разбить hunk на более мелкие части |
| `e` | Открыть hunk в редакторе для ручного редактирования |
| `q` | Выйти, остальные hunks не трогать |
| `a` | Добавить этот и все оставшиеся hunks в файле |
| `d` | Пропустить этот и все оставшиеся hunks в файле |
| `?` | Справка по всем командам |

## Как использовать

```bash
git add -p                       # все изменённые файлы
git add -p src/styles/layout.css # конкретный файл

# После выбора hunks:
git status                       # проверить что в индексе
git diff --staged                # посмотреть что войдёт в коммит
git diff                         # что осталось в worktree
git commit -m "fix: header grid layout"

# Повторить для следующего коммита:
git add -p src/styles/layout.css
git commit -m "refactor: modal z-index system"
```

## Когда `s` не работает

Если hunk нельзя автоматически разбить (изменения слишком близко друг к другу), `s` ответит "Sorry, cannot split this hunk". Тогда используй `e` — откроется редактор с diff-форматом. Удали строки с `-` которые не нужны, замени их на пробел (контекст).

## Подвохи

- **`git add -p` не видит новые файлы.** Для новых файлов — `git add -N <file>` сначала (добавить в индекс как "намерение добавить"), потом `-p`.
- **После `e`** редактируешь diff напрямую: строки с `-` это удаления, с `+` это добавления, с пробелом — контекст. Не удаляй строки контекста.
- **`git checkout -p`** и **`git restore -p`** работают аналогично, но для отмены изменений по кускам.
- **`git stash -p`** — то же, но для stash. Подробнее: [`stash-patch`](./stash-patch.md).
