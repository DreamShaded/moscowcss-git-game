# `git stash -p` — частичный stash по кускам

`git stash -p` (или `--patch`) работает как `git add -p`, но для stash: позволяет выбрать, какие именно изменения спрятать, а какие оставить в worktree.

Это полезно когда в файле несколько несвязанных правок, и нужно спрятать только часть — например, экспериментальный код, оставив рабочие изменения.

## Как использовать

```bash
git stash -p                     # интерактивный выбор hunks для stash
```

Диалог такой же, как в `git add -p`: `y` добавить в stash, `n` оставить, `s` разбить, `q` выйти.

## Подвохи

- **`git stash -p` не берёт untracked файлы.** Только уже отслеживаемые изменения. Для untracked нужен `git stash -u` (но он не поддерживает `-p` одновременно).
- **Частичный stash оставляет файл в "смешанном" состоянии**: часть изменений в stash, часть в worktree. `git status` покажет файл как изменённый.
- **`git stash pop`** применит только то, что было заspashено. Остальные изменения никуда не делись.
- **Альтернатива через `add -p`**: `git add -p` → `git stash --keep-index` спрячет всё что *не* в индексе. Потом `git reset HEAD` для очистки индекса. Иногда это проще.
